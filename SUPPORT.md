# Support

Use GitHub Issues for reproducible bugs, feature requests, and documentation
gaps.

For conduct issues, follow `CODE_OF_CONDUCT.md`. Use public issues only when
public discussion is appropriate.

For undisclosed vulnerabilities or safety bypasses, use the private security
advisory flow described in `SECURITY.md`.

For hardware bring-up questions, include the target MCU, board name, toolchain,
probe, power evidence, command output, and whether `--run-hardware` was used.
