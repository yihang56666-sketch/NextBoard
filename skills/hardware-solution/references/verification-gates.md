# Verification Gates

每个阶段结束前必须通过对应的验证门控。未通过的项不能进入下一阶段。

## Gate 1：需求冻结后

- [ ] 约束表中无空白必填项（场景、供电、通信、环境、成本、认证）。
- [ ] 每个"待确认"项标注了负责人和截止时间。
- [ ] 用户已确认需求表，或明确授权基于假设推进。

## Gate 2：架构选择后

- [ ] 至少比较了两个候选架构（除非用户指定平台）。
- [ ] 每个候选方案有明确的优缺点和适用条件。
- [ ] 推荐方案的选择理由不是"常用"或"熟悉"，而是基于约束匹配。
- [ ] 用户已确认架构方向。

## Gate 3：选型完成后

- [ ] 关键器件（MCU、PMIC、无线、连接器）均有替代方案。
- [ ] 供应链风险已联网查证：生命周期、库存、交期、认证。
- [ ] 风险表非空，且高风险项有验证动作和截止点。
- [ ] 器件参数来自数据手册或分销商页面，非记忆。

## Gate 4：方案输出后

- [ ] 输出包含系统框图、接口矩阵、电源树、PCB 约束、验证计划。
- [ ] 风险清单非空，每个风险有影响、概率、验证动作。
- [ ] 无占位符文本（"待补充""TBD""xxx"）出现在已确定项中。
- [ ] 已确定/待确认/高风险三类事项在末尾明确分列。

## Gate 5：评审完成后

- [ ] 使用 `hardware-reviewer` agent 完成独立评审。
- [ ] 评审发现的 CONCERN 和 FAIL 项已处理或标注为待确认。
- [ ] 最终方案可直接交给硬件工程师开始原理图设计。
