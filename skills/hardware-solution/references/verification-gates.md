# Verification Gates

每个阶段结束前必须通过对应的验证门控。未通过的项不能进入下一阶段。

## Gate 1：需求冻结后

- [ ] 约束表中无空白必填项（场景、供电、通信、环境、成本、认证）。
- [ ] 已向用户提供 2-4 个结构化决策选项，覆盖成本、功耗、交付周期或供应链取向。
- [ ] 每个"待确认"项标注了负责人和截止时间。
- [ ] 用户已确认需求表，或明确授权基于假设推进。

## Gate 2：架构选择后

- [ ] 至少比较了三个候选架构：国产优先、海外主流、混合折中（除非用户指定平台或排除某类）。
- [ ] 每个候选方案有明确的优缺点和适用条件。
- [ ] 每个候选方案覆盖主控、电源、通信接口、传感器/模拟前端的器件范围。
- [ ] 推荐方案的选择理由不是"常用"或"熟悉"，而是基于约束匹配。
- [ ] 用户已确认架构方向。

## Gate 3：选型完成后

- [ ] 关键器件（MCU、PMIC、无线、连接器）均有替代方案。
- [ ] 每个关键模块至少比较了国内候选和海外候选，或说明其中一类不适用的原因。
- [ ] 涉及国产替代或中国市场量产时，已参考 `domestic-sources.md` 覆盖国内主流芯片、元器件、模组和方案平台。
- [ ] Datasheet 来源符合优先级：原厂官网或授权分销商优先；若使用聚合站，已标注为线索并完成交叉验证。
- [ ] 如果参考了方案拆解或参考设计平台，已明确可借鉴点、不可直接复用点，并回溯到原厂资料验证。
- [ ] 供应链风险已联网查证：生命周期、库存、交期、认证。
- [ ] 风险表非空，且高风险项有验证动作和截止点。
- [ ] 器件参数来自数据手册或分销商页面，非记忆。

## Gate 4：方案输出后

- [ ] 输出包含系统框图、接口矩阵、电源树、PCB 约束、验证计划。
- [ ] 风险清单非空，每个风险有影响、概率、验证动作。
- [ ] 无占位符文本（"待补充""TBD""xxx"）出现在已确定项中。
- [ ] 已确定/待确认/高风险三类事项在末尾明确分列。

## Gate 5：评审完成后

- [ ] 使用 `hardware-reviewer` agent 完成独立评审。
- [ ] 评审发现的 CONCERN 和 FAIL 项已处理或标注为待确认。
- [ ] 最终方案可直接交给硬件工程师开始原理图设计。
